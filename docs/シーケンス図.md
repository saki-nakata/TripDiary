## 概要

本ドキュメントでは、主要な操作フローを Mermaid シーケンス図で示す。

### 前提知識：認証セッション

| 項目 | 仕様 |
|------|------|
| 認証方式 | Auth.js v5 セッションベース（Database Strategy） |
| セッション取得 | `auth()` をサーバーサイドで呼び出す |
| セッション保存先 | MySQL（sessions テーブル） |
| セッション有効期限 | 未定（Auth.js のデフォルト設定に従う） |

## 1. ユーザー登録

```mermaid
sequenceDiagram
    actor User
    participant FE as Next.js (フロント)
    participant API as Route Handler
    participant DB as MySQL (Prisma)

    User->>FE: 名前・メール・パスワードを入力して登録ボタン押下
    FE->>FE: バリデーション（必須・文字数・メール形式）
    alt バリデーションエラー
        FE-->>User: エラーメッセージを表示
    else バリデーション OK
        FE->>API: POST /api/auth/signup
        API->>DB: メール重複チェック（SELECT users WHERE email）
        alt メール重複
            DB-->>API: ユーザーが存在する
            API-->>FE: 409 { error: "このメールアドレスはすでに使用されています" }
            FE-->>User: エラーメッセージを表示
        else 重複なし
            API->>API: bcrypt でパスワードをハッシュ化
            API->>DB: ユーザーを INSERT
            DB-->>API: 作成されたユーザー
            API-->>FE: 201 { id, name, email }
            FE->>API: signIn('credentials', ...) を呼び出す
            API->>DB: セッションを INSERT
            API-->>FE: セッション確立
            FE-->>User: フィード画面にリダイレクト
        end
    end
```

## 2. ログイン

```mermaid
sequenceDiagram
    actor User
    participant FE as Next.js (フロント)
    participant Auth as Auth.js
    participant DB as MySQL (Prisma)

    User->>FE: メール・パスワードを入力してログインボタン押下
    FE->>Auth: signIn('credentials', { email, password })
    Auth->>DB: メールでユーザーを検索（SELECT users WHERE email）
    alt ユーザーが存在しない
        DB-->>Auth: null
        Auth-->>FE: CredentialsSignin エラー
        FE-->>User: "メールアドレスまたはパスワードが正しくありません"
    else ユーザーが存在する
        Auth->>Auth: bcrypt.compare(password, hash)
        alt パスワード不一致
            Auth-->>FE: CredentialsSignin エラー
            FE-->>User: "メールアドレスまたはパスワードが正しくありません"
        else パスワード一致
            Auth->>DB: セッションを INSERT
            Auth-->>FE: セッション確立
            FE-->>User: フィード画面にリダイレクト
        end
    end
```

## 3. 認証済み API リクエスト

```mermaid
sequenceDiagram
    actor User
    participant FE as Next.js (フロント)
    participant API as Route Handler
    participant Auth as Auth.js
    participant DB as MySQL (Prisma)

    User->>FE: 操作（いいね・投稿・フォロー等）
    FE->>API: API リクエスト
    API->>Auth: auth() でセッション取得
    alt セッションなし（未認証）
        Auth-->>API: null
        API-->>FE: 401 { error: "Unauthorized" }
        FE-->>User: ログイン画面にリダイレクト
    else セッションあり
        Auth-->>API: session.user（id・name等）
        API->>DB: DB 操作（Prisma）
        DB-->>API: 結果
        API-->>FE: 200 レスポンス
        FE-->>User: UI を更新
    end
```

## 4. ログアウト

```mermaid
sequenceDiagram
    actor User

    participant Auth as Auth.js
    participant DB as MySQL (Prisma)
```

## 5. 投稿作成（写真アップロード含む）

```mermaid
sequenceDiagram
    actor User
    participant FE as Next.js (フロント)
    participant UploadAPI as Route Handler (/api/upload/post)
    participant Cloudinary as Cloudinary
    participant PostAPI as Route Handler (/api/posts)
    participant DB as MySQL (Prisma)

    User->>FE: 写真を選択・フォームを入力して投稿ボタン押下
    FE->>FE: バリデーション（スポット名・感想・エリアタグ必須チェック）
    alt バリデーションエラー
        FE-->>User: エラーメッセージを表示
    else バリデーション OK
        loop 各写真ファイル
            FE->>UploadAPI: POST /api/upload/post (multipart/form-data)
            UploadAPI->>UploadAPI: MIME タイプ・サイズ検証
            UploadAPI->>Cloudinary: 画像をアップロード
            Cloudinary-->>UploadAPI: 画像 URL
            UploadAPI-->>FE: { url: "https://res.cloudinary.com/..." }
        end
        FE->>PostAPI: POST /api/posts { title, body, areaTag, images: [{url, order}] }
        PostAPI->>DB: posts テーブルに INSERT
        PostAPI->>DB: post_images テーブルに INSERT（複数件）
        DB-->>PostAPI: 作成された投稿
        PostAPI-->>FE: 201 { id, title }
        FE-->>User: 投稿詳細画面にリダイレクト
    end
```

## 6. いいね toggle

```mermaid
sequenceDiagram
    actor User
    participant FE as Next.js (フロント)
    participant API as Route Handler (/api/posts/[id]/like)
    participant DB as MySQL (Prisma)

    User->>FE: いいねボタンを押下
    FE->>API: POST /api/posts/[id]/like
    API->>DB: likes テーブルで (userId, postId) を検索
    alt いいねしていない
        DB-->>API: レコードなし
        API->>DB: likes テーブルに INSERT
        DB-->>API: 作成完了
        API-->>FE: { liked: true, likeCount: N+1 }
    else いいね済み
        DB-->>API: レコードあり
        API->>DB: likes テーブルから DELETE
        DB-->>API: 削除完了
        API-->>FE: { liked: false, likeCount: N-1 }
    end
    FE-->>User: ハートアイコンの状態とカウントを更新
```

## 7. プロフィール画像アップロード

```mermaid
sequenceDiagram
    actor User
    participant FE as Next.js (フロント)
    participant UploadAPI as Route Handler (/api/upload/avatar)
    participant Cloudinary as Cloudinary
    participant UserAPI as Route Handler (/api/users/[id])
    participant DB as MySQL (Prisma)

    User->>FE: 設定画面でプロフィール画像を選択
    FE->>UploadAPI: POST /api/upload/avatar (multipart/form-data)
    UploadAPI->>UploadAPI: MIME タイプ・サイズ検証
    UploadAPI->>Cloudinary: 画像をアップロード
    Cloudinary-->>UploadAPI: 画像 URL
    UploadAPI-->>FE: { url: "https://res.cloudinary.com/..." }
    FE->>UserAPI: PUT /api/users/[id] { image: url }
    UserAPI->>DB: users テーブルの image カラムを UPDATE
    DB-->>UserAPI: 更新完了
    UserAPI-->>FE: 200 OK
    FE-->>User: プロフィール画像を即時反映
```
