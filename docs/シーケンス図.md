**更新日:** 2026-06-28

## 概要

本ドキュメントでは、主要な操作フローを Mermaid シーケンス図で示す。

### 前提知識：認証セッション

| 項目 | 仕様 |
|------|------|
| 認証方式 | Auth.js v5 JWT Strategy（HttpOnly Cookie に署名済み JWT を保存） |
| セッション取得 | `auth()` をサーバーサイドで呼び出す |
| セッション保存先 | HttpOnly Cookie（DB への書き込みなし） |
| セッション有効期限 | 30日（アクティブ利用中は毎日自動延長） |

## 1. ユーザー登録

```mermaid
sequenceDiagram
    actor User
    participant FE as Next.js (フロント)
    participant API as Route Handler
    participant DB as MySQL (Prisma)

    User->>FE: 名前・メール・パスワードを入力して登録ボタン押下
    FE->>FE: バリデーション（必須・文字数・メール形式）
    alt バリデーションエラー
        FE-->>User: エラーメッセージを表示
    else バリデーション OK
        FE->>API: POST /api/auth/signup
        API->>DB: メール重複チェック（SELECT users WHERE email）
        alt メール重複
            DB-->>API: ユーザーが存在する
            API-->>FE: 409 { error: "このメールアドレスはすでに使用されています" }
            FE-->>User: エラーメッセージを表示
        else 重複なし
            API->>API: bcrypt でパスワードをハッシュ化
            API->>DB: ユーザーを INSERT
            DB-->>API: 作成されたユーザー
            API-->>FE: 201 { id, name, email }
            FE->>API: signIn('credentials', ...) を呼び出す
            API->>API: JWT を生成し HttpOnly Cookie にセット
            API-->>FE: セッション確立（Cookie）
            FE-->>User: フィード画面にリダイレクト
        end
    end
```

## 2. ログイン

```mermaid
sequenceDiagram
    actor User
    participant FE as Next.js (フロント)
    participant Auth as Auth.js
    participant DB as MySQL (Prisma)

    User->>FE: メール・パスワードを入力してログインボタン押下
    FE->>Auth: signIn('credentials', { email, password })
    Auth->>DB: メールでユーザーを検索（SELECT users WHERE email）
    alt ユーザーが存在しない
        DB-->>Auth: null
        Auth-->>FE: CredentialsSignin エラー
        FE-->>User: "メールアドレスまたはパスワードが正しくありません"
    else ユーザーが存在する
        Auth->>Auth: bcrypt.compare(password, hash)
        alt パスワード不一致
            Auth-->>FE: CredentialsSignin エラー
            FE-->>User: "メールアドレスまたはパスワードが正しくありません"
        else パスワード一致
            Auth->>Auth: JWT を生成し HttpOnly Cookie にセット
            Auth-->>FE: セッション確立（Cookie）
            FE-->>User: フィード画面にリダイレクト
        end
    end
```

## 3. 認証済み API リクエスト

```mermaid
sequenceDiagram
    actor User
    participant FE as Next.js (フロント)
    participant API as Route Handler
    participant Auth as Auth.js
    participant DB as MySQL (Prisma)

    User->>FE: 操作（いいね・投稿・フォロー等）
    FE->>API: API リクエスト
    API->>Auth: auth() でセッション取得
    alt セッションなし（未認証）
        Auth-->>API: null
        API-->>FE: 401 { error: "Unauthorized" }
        FE-->>User: ログイン画面にリダイレクト
    else セッションあり
        Auth-->>API: session.user（id・name等）
        API->>DB: DB 操作（Prisma）
        DB-->>API: 結果
        API-->>FE: 200 レスポンス
        FE-->>User: UI を更新
    end
```

## 5. 投稿作成（写真アップロード含む）

```mermaid
sequenceDiagram
    actor User
    participant FE as Next.js (フロント)
    participant UploadAPI as Route Handler (/api/upload/post)
    participant S3 as S3
    participant PostAPI as Route Handler (/api/posts)
    participant DB as MySQL (Prisma)

    User->>FE: 写真を選択・フォームを入力して投稿ボタン押下
    FE->>FE: バリデーション（スポット名・感想・エリアタグ必須チェック）
    alt バリデーションエラー
        FE-->>User: エラーメッセージを表示
    else バリデーション OK
        loop 各写真ファイル
            FE->>UploadAPI: POST /api/upload/post (multipart/form-data)
            UploadAPI->>UploadAPI: MIME タイプ・サイズ検証
            UploadAPI->>S3: 画像をアップロード
            S3-->>UploadAPI: 画像 URL
            UploadAPI-->>FE: { url: "https://s3.ap-northeast-1.amazonaws.com/tripdiary/..." }
        end
        FE->>PostAPI: POST /api/posts { title, body, areaTag, images: [{url, order}] }
        PostAPI->>DB: posts テーブルに INSERT
        PostAPI->>DB: post_images テーブルに INSERT（複数件）
        DB-->>PostAPI: 作成された投稿
        PostAPI-->>FE: 201 { id, title }
        FE-->>User: 投稿詳細画面にリダイレクト
    end
```

## 6. いいね toggle

```mermaid
sequenceDiagram
    actor User
    participant FE as Next.js (フロント)
    participant API as Route Handler (/api/posts/[id]/like)
    participant DB as MySQL (Prisma)

    User->>FE: いいねボタンを押下
    FE->>API: POST /api/posts/[id]/like
    API->>DB: likes テーブルで (userId, postId) を検索
    alt いいねしていない
        DB-->>API: レコードなし
        API->>DB: likes テーブルに INSERT
        DB-->>API: 作成完了
        API-->>FE: { liked: true, likeCount: N+1 }
    else いいね済み
        DB-->>API: レコードあり
        API->>DB: likes テーブルから DELETE
        DB-->>API: 削除完了
        API-->>FE: { liked: false, likeCount: N-1 }
    end
    FE-->>User: ハートアイコンの状態とカウントを更新
```

## 7. プロフィール画像アップロード

```mermaid
sequenceDiagram
    actor User
    participant FE as Next.js (フロント)
    participant UploadAPI as Route Handler (/api/upload/avatar)
    participant S3 as S3
    participant UserAPI as Route Handler (/api/users/[id])
    participant DB as MySQL (Prisma)

    User->>FE: 設定画面でプロフィール画像を選択
    FE->>UploadAPI: POST /api/upload/avatar (multipart/form-data)
    UploadAPI->>UploadAPI: MIME タイプ・サイズ検証
    UploadAPI->>S3: 画像をアップロード
    S3-->>UploadAPI: 画像 URL
    UploadAPI-->>FE: { url: "https://s3.ap-northeast-1.amazonaws.com/tripdiary/..." }
    FE->>UserAPI: PUT /api/users/[id] { image: url }
    UserAPI->>DB: users テーブルの image カラムを UPDATE
    DB-->>UserAPI: 更新完了
    UserAPI-->>FE: 200 OK
    FE-->>User: プロフィール画像を即時反映
```

## 8. 旅行プラン作成

```mermaid
sequenceDiagram
    actor User
    participant FE as Next.js (フロント)
    participant API as Route Handler
    participant DB as MySQL (Prisma)

    User->>FE: タイトル・日程・スポット・予算を入力して「プランを作成する」押下
    FE->>FE: バリデーション（タイトル必須・終了日 ≥ 開始日）
    alt バリデーションエラー
        FE-->>User: エラーメッセージを表示
    else バリデーション OK
        FE->>FE: 送信ボタンを無効化（二重送信防止）
        FE->>API: POST /api/plans { title, startDate, endDate, budgetBreakdown, memo, spotIds }
        API->>API: セッション確認（未認証なら 401）
        API->>API: budgetBreakdown から budget 合計を計算
        API->>DB: plans テーブルに INSERT
        API->>DB: plan_spots テーブルに spotIds を displayOrder 付きで INSERT
        DB-->>API: 作成されたプラン
        API-->>FE: 201 { id, title, ... }
        FE-->>User: トースト表示 → プラン詳細画面へ遷移
    end
```

## 9. 「この旅を記録する」フロー

```mermaid
sequenceDiagram
    actor User
    participant PlanDetail as プラン詳細画面
    participant PostNew as 投稿作成画面
    participant API as Route Handler
    participant DB as MySQL (Prisma)

    User->>PlanDetail: 「この旅を記録する →」をクリック
    PlanDetail->>PostNew: 遷移（fromPlan・presetTitle・presetAreaTag・presetCategory をクエリパラメータで渡す）
    PostNew->>PostNew: フォームにプリセット値を反映
    User->>PostNew: 感想・写真・費用内訳などを入力して「投稿する」押下
    PostNew->>API: POST /api/posts { title, body, ..., planId: fromPlan }
    API->>DB: posts テーブルに planId を含めて INSERT
    DB-->>API: 作成された投稿
    API-->>PostNew: 201 { id, ... }
    PostNew-->>User: トースト表示 → プラン詳細画面へリダイレクト
    Note over PlanDetail: プラン詳細の「このプランから作成した投稿」セクションに投稿が表示される
```

## 10. 旅行レポート取得

```mermaid
sequenceDiagram
    actor User
    participant FE as Next.js (フロント)
    participant YearsAPI as Route Handler (/api/stats/years)
    participant StatsAPI as Route Handler (/api/stats)
    participant DB as MySQL (Prisma)

    User->>FE: マイページ「旅行レポート」タブを開く
    FE->>YearsAPI: GET /api/stats/years
    YearsAPI->>DB: 自分の投稿の visitedAt を全件取得
    DB-->>YearsAPI: visitedAt リスト
    YearsAPI->>YearsAPI: 年を抽出・重複排除・降順ソート
    YearsAPI-->>FE: { years: [2026, 2025, 2024] }
    FE-->>User: 年選択ドロップダウンを表示（最新年をデフォルト選択）

    User->>FE: 年選択ドロップダウンで年を選択
    FE->>StatsAPI: GET /api/stats?year=2026
    StatsAPI->>DB: 対象年の自分の Post・PostImage を取得
    StatsAPI->>DB: 対象年の自分の完了済み Plan を取得
    DB-->>StatsAPI: Post・PostImage・Plan データ
    StatsAPI->>StatsAPI: 集計処理（JS側）<br/>- completedPlans カウント<br/>- prefecture ユニーク抽出・最頻値<br/>- totalPhotos 合計<br/>- totalCost 合計<br/>- カテゴリ別件数<br/>- 月別費用合計
    StatsAPI-->>FE: StatsResponse（集計済みデータ）
    FE-->>User: Wrapped風カード・グラフ・タイムラインを表示
```
