### S-03 探索（全体フィード）画面

**URL：** `/`（未認証・認証済み両方に同一内容を表示。ログイン済みでもこの画面がホームとして機能し、別画面 `/dashboard` は単なるリダイレクトのみ）

```
+------------------+--------------------------------+
|   サイドバー      |  ❤️ 人気の旅スポット            |
|                  |  [投稿カードグリッド 4列]        |
|                  |                                |
|                  |  📍 エリアから探す              |
|                  |  [エリアカード一覧（画像+件数）]|
|                  |                                |
|                  |  🗂️ カテゴリから探す            |
|                  |  [カテゴリカード一覧]            |
|                  |                                |
|                  |  ⭐ カテゴリ別高評価スポット     |
|                  |  [カテゴリ×最高評価投稿カード]   |
|                  |                                |
|                  |  🆕 新着の旅スポット            |
|                  |  [投稿カードグリッド 4列]        |
+------------------+--------------------------------+
```

| 要素 | 内容 |
|------|------|
| 人気の旅スポット | いいね数降順で上位6件をカードグリッド表示 |
| エリアから探す | 投稿が存在するエリアをサムネイル画像・件数付きカードで一覧表示（`AreaSection.tsx`）。「エリア検索」リンクから `/search?tab=area` へ遷移可能 |
| カテゴリから探す | カテゴリ別にアイコン・件数付きカードで表示（`CategorySection.tsx`）。現状クリックしても絞り込み画面はない |
| カテゴリ別高評価スポット | 各カテゴリの中で評価（rating）が最も高い投稿を1件ずつカード表示。同評価の場合はいいね数降順。「人気の旅スポット」に含まれる投稿は除外 |
| 新着の旅スポット | 新着順（createdAt DESC）で上位6件をカードグリッド表示 |
| 対象 | 全ユーザーの投稿（未認証でも表示、認証済みユーザーも同じ内容） |

> 「もっと見る」リンク（人気順は`/search?tab=post&sort=popular`、新着順は`/search?tab=post`への遷移）は Phase 3-D の `/search` 実装に伴い有効化済み。

**データ取得方式：** `page.tsx`（Server Component）が初回SSR用にrepository関数を直接呼び出し、その結果を`initialData`としてクライアントコンポーネント`ExploreFeed.tsx`に渡す。`ExploreFeed.tsx`はTanStack Queryの`useQuery`（`queryKey: ["explore-feed"]`、`staleTime: 180_000`、`refetchInterval: 60_000`）で`GET /api/posts/portal`をポーリングし、60秒ごとに全セクションを更新する。

**新規投稿後のスクロール＆ハイライト：** 投稿作成（`/posts/new`）成功時、`PostForm.tsx`が`queryClient.setQueryData`で「新着の旅スポット」の先頭に新規投稿を即座に追加した上で`/?highlighted=<投稿id>`へ遷移する。本画面に到達すると`HighlightedPostScroller`コンポーネントが該当投稿カード（`data-post-id`属性で照合。同じ投稿が複数セクションに重複表示され得るため、DOM上で最後に見つかった要素＝新着セクションを対象にする）までスムーズスクロールし、2秒間ハイライト表示した後にURLの`?highlighted=`パラメータを除去する。「人気の旅スポット」「カテゴリ別高評価スポット」はいいね数・評価順のランキングに依存するため、新規投稿がここに表示される保証はない。

### S-10 マイページ

**URL：** `/mypage`（認証必須）

ページ上部に「🔒 この画面は自分のみ表示」バナーを表示。タブ切り替えで各コンテンツを表示する。

```
+------------------+--------------------------------+
|   サイドバー      |  [タイトル]  🔒この画面は自分のみ表示|
|                  |                                |
|                  |  [旅行プラン][自分の投稿][旅行レポート][行きたい][訪問済み][フォロー中の投稿] |
|                  |                                |
|                  |  （選択タブのコンテンツ）        |
+------------------+--------------------------------+
```

| タブ | 内容 |
|------|------|
| 🗺️ 旅行プラン | 進行中プランのカード一覧 + 完了済みプランのアコーディオン。カウントバッジは完了済みを除外。アコーディオン内に完了済みプランの年度切り替えセレクトを表示（開始日=`startDate`の年で絞り込み、2026-07-20追加）。デフォルト表示タブ |
| ✈️ 自分の投稿 | 自分が投稿したスポットのカードグリッド。年度切り替えセレクトあり（2026-07-19追加、年度に応じてカウントバッジも変動）。各カードにホバーで ✏️（編集）・🗑️（削除）アイコンを表示。費用・費用内訳の表示は行わない（2026-07-20削除） |
| 📋 旅行レポート | 年別の旅まとめカード・エリア別バブルチャート・高評価スポットTOP3・カテゴリ別バーグラフ・月別ヒートマップ・（全期間表示時のみ）年別投稿数の推移を表示。年度切り替えセレクトは本タブヘッダー行に統合（「自分の投稿」と共通コンポーネント）。詳細は S-17 参照（2026-07-20全面改訂） |
| 🔖 行きたい | 都道府県マップ（登録件数に応じた濃淡）＋県別グループのカード一覧。地図クリックで該当グループへスクロール、各カードから「外す」ボタンで登録解除できる。詳細は S-19 参照（2026-07-20刷新、旧仕様は単純なカードグリッド） |
| ✅ 訪問済み | 行きたいタブと同じ地図＋グループ一覧UI（詳細は S-19 参照）。2026-07-20刷新、旧仕様は単純なカードグリッド |
| 👥 フォロー中の投稿 | フォロー中ユーザーの投稿を「今日/昨日/今週/今月/それ以前」の日付グループに分けたSNS風フィード（投稿者アイコン・相対時刻・エリアバッジ付き）。列数はスマホ1列・タブレット2列・PC4列。詳細は S-20 参照（2026-07-20刷新、旧仕様は単純なカードグリッド） |
