## 1. 機能概要

自分の旅行記録を年別に集計・可視化する振り返り機能。
Spotify Wrapped 風の年間まとめカードをはじめ、エリア別の広がり・高評価スポットのハイライト・カテゴリ分析・活動頻度のヒートマップ・投稿数の推移を提供する。
費用に関する集計・可視化は行わない方針（2026-07-20変更）。
4Travel にはない付加価値として、旅行記録アプリならではの統計体験を実現する。

## 3. 機能詳細

### 3.1 年間旅行まとめカード（Wrapped 風）

- URL：`/mypage?tab=report`（年選択時は `/mypage?tab=report&year=YYYY`）
- 年選択ドロップダウンで対象年を切り替え（投稿がある年に加えて「全期間」を選択肢の先頭に表示。初期表示は「全期間」）
- 年セレクター（`YearFilterBar`）は`mypage/page.tsx`のページヘッダーに配置し、「自分の投稿」タブと共通のコンポーネントを使う（2026-07-20変更：旧仕様は本コンポーネント内に独自の年セレクターを持っていた）。選択すると `/mypage?tab=report&year=YYYY` へ`router.replace()`される。スマホでは「🔒 この画面は自分のみ表示」バッジと同じ行に左寄せ表示し、「表示する年：」ラベルは非表示（スマホのみ）
- 選択年（または全期間）の投稿・プランデータを集計し、以下のカードをグラデーション背景でアニメーション表示。単年表示時は7枚、全期間表示時は9枚（2026-07-20変更：旧8枚から再構成）
- カードは `animation-delay` を 0.1s ずつずらして順番にフェードインアニメーション

| カード | 集計対象 | クリック時の遷移先 |
|-------|---------|------|
| 投稿数 | 選択年（全期間の場合は全年）の Post 件数 | 3.5節 月別ヒートマップ |
| 写真枚数 | 選択年（全期間の場合は全年）の Post に紐付く PostImage の合計枚数 | 自分の投稿一覧 |
| 訪問エリア数 | 選択年（全期間の場合は全年）の Post の `location` ユニーク件数 | 3.2節 エリア別バブルチャート |
| 最多訪問エリア | `location` の最頻値（同数の場合はエリア別バブルチャートの先頭＝件数降順・同数は都道府県順で最初のもの。2026-07-20変更：旧仕様は初回出現順だったが、バブルと矛盾しないよう基準を統一） | 訪問済み一覧 |
| カテゴリ数 | 選択年（全期間の場合は全年）の `categoryBreakdown` の件数（投稿したカテゴリの種類数） | 3.4節 カテゴリ別投稿数 |
| 旅行回数 | `completed=true` かつ `startDate` が選択年（全期間の場合は全年）の Plan 件数 | 旅行プラン一覧 |
| 平均評価 | 選択年（全期間の場合は全年）の評価済み投稿の★平均（2026-07-20追加。「カテゴリ数」「旅行費用合計」カードを廃した代わりに追加） | 3.3節 高評価スポットTOP3 |
| 旅歴（全期間表示時のみ） | 投稿がある年の数（`years.length`） | 3.5節 月別ヒートマップ |
| 最も旅した年（全期間表示時のみ） | `yearlyPostCount`が最大の年（同数タイは新しい年を優先） | 3.5節 月別ヒートマップ |

- 「旅行費用合計」カードは2026-07-20に削除（費用の可視化を行わない方針への変更）

### 3.2 エリア別バブルチャート

- 見出しは「{yearLabel}のエリア別投稿数」（例: 「2026年のエリア別投稿数」「全期間のエリア別投稿数」）
- 選択年（全期間の場合は全年）の Post を `location` でグルーピングし件数を集計（`location`未設定の投稿は除外）
- 件数の多いエリアほど大きな円（バブル）で表示する。円のサイズは件数の平方根スケール（52px〜116px）、色は色相を大きく振った12色から件数順にローテーション割り当て
- 各バブルはクリックすると検索ページのエリアタブ（`/search?tab=area&location=<エリア名>`）へ遷移する（2026-07-20追加。回遊導線の確保）
- 年度切り替え時は再マウントしてポップイン＋浮上アニメーションを再生する
- スマホ（640px未満）のみ`--bubble-scale`CSS変数で全体を0.75倍に縮小し、文字サイズも連動して縮む
- 2026-07-20変更：旧仕様のバッジ一覧＋日本地図の塗り分け表示（`PrefectureMapView`）はこのタブから削除。地図での可視化は「行きたい」「訪問済み」タブに残っている（別コンポーネント、影響なし）

### 3.3 高評価スポットTOP3

- 見出しは「{yearLabel}の高評価スポットTOP3」
- 選択年（全期間の場合は全年）の投稿から、★評価が設定されている投稿を評価降順（同点は訪問日が新しい方を優先）で上位3件抽出
- 評価が1件もない場合はブロック自体を非表示
- 各カードはサムネイル画像＋左上に順位バッジ（🥇🥈🥉）＋右上に★評価バッジ、下部にタイトル・エリア（緑文字）・カテゴリ（アイコン＋グレー文字）・訪問日（右端）を表示
- カードクリックで投稿詳細へ遷移
- 2026-07-20追加：旧仕様の「評価の分布」（★1〜5件数の横棒グラフ）を置き換える形で新設。評価が★4〜5に偏りがちで棒グラフでは差が見えにくいという課題への対応

### 3.4 カテゴリ別投稿数（CSS バーグラフ）

- 見出しは「{yearLabel}のカテゴリ別投稿数」（例: 「2026年のカテゴリ別投稿数」「全期間のカテゴリ別投稿数」）
- 選択年（全期間の場合は全年）の Post を `category` でグルーピングし件数を集計
- 件数降順で CSS バーグラフ（横棒）として表示
- 最多カテゴリを 100% として各カテゴリの幅を相対表示
- バーの配色はバイオレット（左側85%）→明るいピンク（右端15%）のグラデーション

### 3.5 月別ヒートマップ（投稿数）

- 見出しは「{year}年の月別投稿数」（単年）／「全期間の月別投稿数（季節性）」（全期間）
- 特定の年を選択している場合：選択年の Post を訪問月（`visitedAt` の月）でグルーピングした月別件数を1〜12月すべて表示（投稿がない月は0件）
- 「全期間」を選択している場合：全 Post を年をまたいで「その月（1〜12月）」に何件旅したか通算した季節性の集計を表示
- `grid-cols-6 sm:grid-cols-12`のカレンダー風グリッドで、セルの色は淡いピンク→明るいピンクの単一色相・逐次スケール（3段階、セル単体も左上→右下の対角グラデーション）。0件のセルはグレー表示
- 各セルに`title`/`aria-label`（例:「3月：5件」）を設定し、下部に「少ない→多い」の凡例を表示
- 2026-07-20新設。投稿数の推移（折れ線・3.6節）とは軸が異なる（ヒートマップ＝分布の一覧性、折れ線＝時系列の増減トレンド）ため重複しないと判断

### 3.6 年別投稿数の推移（折れ線グラフ）

- 見出しは「全期間の年別投稿数の推移」
- **「全期間」を選択している場合のみ表示**（単年表示時は3.5節の月別ヒートマップが同じ役割を果たすため非表示。2026-07-20変更：旧仕様は単年でも月別推移を折れ線で表示していた）
- 全 Post を訪問年（`visitedAt` の年）でグルーピングし件数を集計した年別データを、SVGハンドロールの折れ線グラフで表示
- 折れ線はY軸目盛り（件数）付き。ポイントは明るいピンク塗り・明るい黄色の枠線（2026-07-20変更）
- 年別ラベルは各データ点の直下に表示する

## 4. 使用 API

| Method | エンドポイント | 説明 | 認証 |
|--------|--------------|------|------|
| GET | `/api/stats/years` | 投稿がある年一覧取得 | 必要 |
| GET | `/api/stats?year=YYYY\|all` | 年別統計データ取得（`year=all` で全期間集計） | 必要 |

**GET `/api/stats?year=YYYY|all` レスポンス型**（2026-07-20更新）

```typescript
type StatsResponse = {
  year: number | "all"
  completedPlans: number
  totalPosts: number
  totalPhotos: number
  totalCost: number                      // 算出はするがUIには表示しない（費用の可視化を行わない方針）
  visitedLocations: string[]
  topLocation: string | null             // locationBreakdown先頭（件数降順・同数は都道府県順）から導出
  locationBreakdown: { location: string; count: number }[]   // エリア別バブルチャート用（3.2節）
  categoryBreakdown: { category: string; count: number }[]
  ratingBreakdown: { rating: number; count: number }[]       // averageRatingの算出のみに使用
  averageRating: number | null
  topRatedPosts: {                       // 高評価スポットTOP3用（3.3節）
    id: string
    title: string
    location: string
    category: string | null
    visitedAt: string
    rating: number
    thumbnail: string | null
  }[]
  monthlyPostCount: { month: number; count: number }[]    // 特定の年を選択した場合のみ値を持つ（月別ヒートマップ用）
  yearlyPostCount: { year: number; count: number }[]       // 全期間を選択した場合のみ値を持つ（折れ線・最も旅した年カード用）
  seasonalPostCount: { month: number; count: number }[]    // 全期間を選択した場合のみ値を持つ（月別ヒートマップ用、年をまたいだ月別通算）
}
```

## 5. バリデーション仕様

| 項目 | 内容 |
|------|------|
| 年選択 | 自分の投稿が 1 件以上存在する年に加えて「全期間」を選択肢に表示（初期選択は「全期間」） |
| 投稿ゼロ時 | 「{yearLabel}の旅の記録がありません。」メッセージを表示 |
| location 未設定 | エリア集計・エリア別バブルチャート表示から除外（エラーにしない） |
| cost 未設定 | `totalCost`の算出では0として集計するが、UI表示自体を行わないため実害なし |
| rating 未設定 | 平均評価・高評価スポットTOP3の集計から除外（エラーにしない） |

---

## 6. 制約・前提条件

- 閲覧できるのは自分のデータのみ（他ユーザーのレポートは非公開）
- `location` は投稿作成・編集時にドロップダウンで選択する必須項目
- 旅行回数の集計は Plan の `completed` フラグと `startDate` の年で判定する。`startDate` 未設定のプランは集計から除外
- 高評価スポットTOP3は同点の場合、訪問日（`visitedAt`）が新しい投稿を優先する

---

## 7. 対象外仕様

| 機能 | 理由 |
|------|------|
| 他ユーザーのレポート閲覧 | プライベートな統計情報のためスコープ外 |
| SNS シェア（画像書き出し） | スコープ外 |
| 費用に関する集計・グラフ（月別/年次比較） | 2026-07-20の方針変更により対象外。振り返り体験としての価値よりも、費用というデリケートな情報をSpotify Wrapped風の演出で見せることの是非を優先した判断 |
| 日本地図の塗り分け表示（本タブ） | 2026-07-20にエリア別バブルチャートへ置き換えたため対象外。地図表示自体は「行きたい」「訪問済み」タブに存続 |
| 年別タイムライン（日付別投稿一覧） | 2026-07-20に削除。「自分の投稿」タブの年度絞り込みと役割が重複するため |
| 累積訪問エリア数・カテゴリの多様性・写真枚数の推移（折れ線の代替案） | 検討したが、既存ブロックとの内容重複や「累積の意味が薄い」等の理由で不採用。最終的に年別投稿数の推移を採用 |
