## 1. 機能概要

自分の旅行記録を年別に集計・可視化する振り返り機能。
Spotify Wrapped 風の年間まとめカードをはじめ、カテゴリ分析・費用グラフ・訪問エリア一覧・年別タイムラインを提供する。
4Travel にはない付加価値として、旅行記録アプリならではの統計体験を実現する。

---

## 2. ユーザーストーリー

| ID | ストーリー |
|----|-----------|
| US-01 | ユーザーとして、今年の旅行を一枚のまとめカードで振り返りたい |
| US-02 | ユーザーとして、何県を訪れたか・何枚写真を撮ったかを数字で確認したい |
| US-03 | ユーザーとして、過去の年の旅行もドロップダウンで切り替えて確認したい |
| US-04 | ユーザーとして、自分がどのカテゴリ（グルメ・自然など）をよく訪れるか傾向を知りたい |
| US-05 | ユーザーとして、月ごと・年ごとの投稿数の推移をグラフで確認したい |
| US-06 | ユーザーとして、全年の旅行記録を年別タイムラインで一覧したい |

## 3. 機能詳細

### 3.1 年間旅行まとめカード（Wrapped 風）

- URL：`/mypage?tab=report`
- 年選択ドロップダウンで対象年を切り替え（投稿がある年に加えて「全期間」を選択肢の先頭に表示。初期表示は「全期間」）
- 選択年（または全期間）の投稿・プランデータを集計し、以下 7 枚のカードをグラデーション背景でアニメーション表示

| カード | 集計対象 |
|-------|---------|
| 旅行回数 | `completed=true` かつ `startDate` が選択年（全期間の場合は全年）の Plan 件数 |
| 訪問エリア数 | 選択年（全期間の場合は全年）の Post の `location` ユニーク件数 |
| 訪問スポット数 | 選択年（全期間の場合は全年）の Post 件数 |
| 投稿数 | 選択年（全期間の場合は全年）の Post 件数（訪問スポット数と同値） |
| 写真枚数 | 選択年（全期間の場合は全年）の Post に紐付く PostImage の合計枚数 |
| 旅行費用合計 | 選択年（全期間の場合は全年）の Post の `cost` 合計（記録なしの場合は「—」） |
| 最多訪問エリア | `location` の最頻値（同数の場合は最初に出現したもの） |

- カードは `animation-delay` を 0.1s ずつずらして順番にフェードインアニメーション

### 3.2 カテゴリ別投稿数（CSS バーグラフ）

- 選択年（全期間の場合は全年）の Post を `category` でグルーピングし件数を集計
- 件数降順で CSS バーグラフ（横棒）として表示
- 最多カテゴリを 100% として各カテゴリの幅を相対表示

### 3.3 投稿数の推移（折れ線グラフ）

- 特定の年を選択している場合：選択年の Post を訪問月（`visitedAt` の月）でグルーピングし件数を集計する「月別投稿数の推移」を表示
  - 1月〜12月を全て表示（投稿がない月は 0 件）
- 「全期間」を選択している場合：全 Post を訪問年（`visitedAt` の年）でグルーピングし件数を集計する「年別投稿数の推移」を表示
- いずれもピンクグラデーションの折れ線グラフ（SVG、Reactハンドロール実装）で表示し、右側にY軸目盛り（件数）と横方向のグリッド線を表示する
- 月別/年別ラベルは各データ点の直下（SVG上のx座標に対応する位置）に表示する

### 3.4 訪問エリアバッジ一覧

- 選択年（全期間の場合は全年）の Post の `location` をユニークで抽出してバッジ形式で表示
- `location` が未設定の投稿は集計から除外

### 3.5 年別タイムライン

- **全年**の自分の投稿を `visitedAt` の年でグルーピング
- 各年セクションは `visitedAt` 降順（新しい訪問日が上）で表示
- **各年 3 件まで表示**し、4 件目以降は「▼ 残りN件を表示」ボタンでトグル展開
- 「▲ 折りたたむ」で再び 3 件に戻せる
- 各行のサムネイル・タイトル・エリア・訪問日をクリックで投稿詳細へ遷移

## 4. 使用 API

| Method | エンドポイント | 説明 | 認証 |
|--------|--------------|------|------|
| GET | `/api/stats/years` | 投稿がある年一覧取得 | 必要 |
| GET | `/api/stats?year=YYYY\|all` | 年別統計データ取得（`year=all` で全期間集計） | 必要 |

**GET `/api/stats?year=YYYY|all` レスポンス型**

```typescript
type StatsResponse = {
  year: number | "all"
  completedPlans: number
  totalPosts: number
  totalPhotos: number
  totalCost: number
  visitedLocations: string[]
  topLocation: string | null
  categoryBreakdown: { category: string; count: number }[]
  monthlyPostCount: { month: number; count: number }[]   // 特定の年を選択した場合のみ値を持つ（全期間選択時は空配列）
  yearlyPostCount: { year: number; count: number }[]      // 全期間を選択した場合のみ値を持つ（特定の年選択時は空配列）
}
```

## 5. バリデーション仕様

| 項目 | 内容 |
|------|------|
| 年選択 | 自分の投稿が 1 件以上存在する年に加えて「全期間」を選択肢に表示（初期選択は「全期間」） |
| 投稿ゼロ時 | 「YYYY年の旅の記録がありません。」（全期間選択時は「全期間の旅の記録がありません。」）メッセージを表示 |
| location 未設定 | エリア集計・訪問エリア表示から除外（エラーにしない） |
| cost 未設定 | 費用合計を 0 として集計（カード表示は「—」） |

---

## 6. 制約・前提条件

- 閲覧できるのは自分のデータのみ（他ユーザーのレポートは非公開）
- `location` は投稿作成・編集時にドロップダウンで選択する必須項目
- 旅行回数の集計は Plan の `completed` フラグと `startDate` の年で判定する。`startDate` 未設定のプランは集計から除外
- タイムラインは選択年に関わらず全年分を表示する（年ドロップダウンの切り替えはまとめカード・グラフ・エリアバッジのみに影響）

---

## 7. 対象外仕様

| 機能 | 理由 |
|------|------|
| 日本地図の塗り分け表示 | プロトタイプではテキストタグで代替。本実装時に検討 |
| 他ユーザーのレポート閲覧 | プライベートな統計情報のためスコープ外 |
| SNS シェア（画像書き出し） | スコープ外 |
| 費用の年次比較グラフ | 「全期間」選択時は年別投稿数の推移グラフを提供する方針とし、費用の年次比較は対象外 |
