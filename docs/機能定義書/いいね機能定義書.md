# いいね機能定義書

**バージョン:** 1.0
**作成日:** 2026-06-27

---

## 1. 機能概要

投稿に対して「いいね（共感・応援）」のリアクションを行う機能。
1ユーザー1投稿につき1回のみ登録でき、同じ API を呼び出すことでトグル（登録・解除）する。
「行きたい」リストへの保存とは独立した機能として管理する（いいね = 共感・応援、行きたい = 自分用ブックマーク）。

---

## 2. ユーザーストーリー

| ID | ストーリー |
|----|-----------|
| US-01 | ユーザーとして、気に入った旅行スポットに「いいね」で共感・応援を伝えたい |
| US-02 | ユーザーとして、いいねを取り消したい |
| US-03 | ユーザーとして、投稿のいいね数を確認したい |
| US-04 | ユーザーとして、自分がいいねした投稿かどうかをひと目で判別したい |

---

## 3. 機能詳細

### 3.1 いいね操作

- `POST /api/posts/[id]/like` を呼び出すことでトグル動作する
  - いいねしていない状態 → いいね登録
  - いいね済みの状態 → いいね解除
- 処理はサーバーサイドで Prisma の upsert / delete で行う

### 3.2 ボタン表示仕様

| 状態 | 表示 |
|------|------|
| 未いいね | 👍 アイコン + いいね数 |
| いいね済み | 👍 アイコン（ハイライト）+ いいね数 |

- ボタン押下時は即座に UI を更新する（楽観的更新：未定、実装時に検討）

### 3.3 いいね数表示

- 投稿カード（フィード）と投稿詳細画面の両方に表示する
- いいね数は `likes` テーブルの件数を都度集計する（非正規化カウンタは未定）

---

## 4. 使用 API

| Method | エンドポイント | 説明 | 認証 |
|--------|--------------|------|------|
| POST | `/api/posts/[id]/like` | いいね toggle（登録 / 解除） | 必要 |

**レスポンス例**

```json
{
  "liked": true,
  "likeCount": 12
}
```

---

## 5. バリデーション仕様

| 項目 | 仕様 |
|------|------|
| 未認証ユーザーのいいね | 401 を返す |
| 重複いいね | toggle 処理のため重複は発生しない |
| 自分の投稿へのいいね | 制限なし（自分の投稿にもいいね可能） |

---

## 6. 制約・前提条件

- 1ユーザーが同一投稿にいいねできるのは1回のみ（DB の複合主キーで保証）
- いいねはログインユーザーのみ操作可能

---

## 7. 対象外仕様

| 機能 | 理由 |
|------|------|
| いいねしたユーザーの一覧表示 | スコープ外 |
| いいね通知 | スコープ外 |
