# 地図機能定義書

**バージョン:** 1.2
**作成日:** 2026-06-27
**更新日:** 2026-07-20

> ✅ **2026-07-20 更新：** 本書はこれまで投稿詳細のピン地図（Leaflet、座標ベース）のみを対象としていたが、**性質の異なるもう一つの地図（都道府県の塗り分け表示、`PrefectureMapView.tsx`、SVGベース・座標不要）**が3.3節時点（2026-07-13、旅行レポート機能）から存在し、本書に未記載だった。3.6節として追記し、対象外仕様（7章）の「複数スポットの地図一覧表示」も実装済みとして訂正した。

---

## 1. 機能概要

投稿した旅行スポットの位置情報を地図上に表示する機能。
Leaflet + OpenStreetMap を使用し、投稿詳細画面にスポットの位置をピンで表示する。

---

## 2. ユーザーストーリー

| ID | ストーリー |
|----|-----------|
| US-01 | ユーザーとして、投稿詳細画面でスポットの場所を地図で確認したい |
| US-02 | ユーザーとして、投稿作成時にスポットの位置（緯度/経度）を設定したい |

---

## 3. 機能詳細

### 3.1 地図表示（投稿詳細）

- 投稿の `lat` / `lng` が設定されている場合のみ地図を表示する（地図拡大ボタンで地図であることが自明なため「🗺️ 地図」の見出しラベルは表示しない）
- 未設定の場合は地図エリア自体を非表示にする
- Leaflet + OpenStreetMap を使用してインタラクティブな地図を表示する
- スポットの位置にピンマーカーを表示する
- 初期ズームレベル：14
- 地図右上の「🗺️地図拡大表示🔍」ボタンから、画面全体を覆うモーダル表示（`w-full max-w-4xl h-[80vh]`）で拡大した地図を確認できる。ESCキーまたは背景クリック、右上の✕ボタンで閉じる

### 3.2 位置情報の設定（投稿作成・編集）

- 入力方式：地図上クリックでピンを立てる（案 B 採用）
- 位置情報の設定は任意とする（未設定でも投稿可能）
- クリック後、再クリックでピンを上書き移動できる
- ピン横の ✕ ボタンでピンと住所表示をリセットできる

### 3.3 逆ジオコーディング（住所表示）

- ピン設置後、Nominatim API で日本語住所を自動取得して地図の**下**に表示する
- エンドポイント：`https://nominatim.openstreetmap.org/reverse?lat={lat}&lon={lng}&format=json&accept-language=ja`
- 取得中は「取得中…」を表示し、取得完了後に `display_name` を表示する（例: `📍 千代田区, 東京都, 日本`）
- 取得失敗時は座標（`📍 lat, lng`）にフォールバックする
- 住所は表示用途のみ。DB には座標（lat/lng）のみ保存する

### 3.4 複数スポットのピン地図（プラン詳細）

- `/plans/[id]` にプランの全スポット（緯度経度がある投稿のみ）をピン留めした地図を表示する（`PlanMapView.tsx`、Leaflet + `react-leaflet`、動的import）
- 複数スポットは `L.latLngBounds` で自動フィットする
- 座標未設定のスポットは地図上に表示されない（対象外）

### 3.5 都道府県塗り分け地図（座標不要、2026-07-20追記）

3.1〜3.4節のLeafletピン地図とは別の、**座標（lat/lng）を使わない**もう一つの地図表示方式。`location`（都道府県名の文字列）を`PrefectureMapView.tsx`（自作SVGコンポーネント）で塗り分けるだけの軽量な可視化で、Leaflet・OpenStreetMap・Nominatimのいずれにも依存しない。

- 実装：Geolonia提供の都道府県別SVGパス（`data-code="1"`〜`"47"`）を使用し、`data-code`の並び順が`src/lib/constants.ts`の`LOCATIONS`配列（北海道〜沖縄県）と一致することを利用して県名と対応付ける
- 塗り分けの濃淡は「その県に何件あるか」（`intensityByLocation`）に応じて変化させられる（件数が多いほど濃い色）
- クリックすると、呼び出し元が指定したコールバック（`onPrefectureClick`）が発火する。新規APIやDB変更は不要（既存データをクライアント側で可視化するのみ）
- **利用箇所**：
  - マイページの「行きたい」「訪問済み」タブ（`SavedMapSection.tsx`）：登録件数で濃淡表示し、クリックで該当県のカード一覧へスクロール（詳細は`リスト機能定義書.md`3.3節）
  - 旅行レポートタブでは2026-07-20に廃止し、エリア別バブルチャートに置き換えた（`旅行レポート機能定義書.md`参照）

### 3.6 ライブラリ構成

| 項目 | 内容 |
|------|------|
| 地図ライブラリ | Leaflet 1.9.4（Next.js では `react-leaflet` + 動的 import）。都道府県塗り分け地図（3.5節）は自作SVGのため対象外 |
| タイル | OpenStreetMap（APIキー不要・無料） |
| 逆ジオコーディング | Nominatim（APIキー不要・無料） |

## 4. 使用 API

| Method | エンドポイント | 説明 | 認証 |
|--------|--------------|------|------|
| （外部）| Leaflet + OpenStreetMap | 地図タイル表示 | 不要 |
| （外部）| Nominatim `/reverse` | 逆ジオコーディング（座標→日本語住所） | 不要 |

※ サーバーサイドの Route Handler は地図機能専用には設けない。座標データは投稿 API（`POST /api/posts`）で一緒に送受信する。

---

## 5. バリデーション仕様

| 項目 | 仕様 |
|------|------|
| 緯度 | 任意・-90 〜 90 の範囲（`postSchema`の`lat`にZodバリデーション実装済み） |
| 経度 | 任意・-180 〜 180 の範囲（`postSchema`の`lng`にZodバリデーション実装済み） |

---

## 6. 制約・前提条件

- Leaflet + OpenStreetMap を使用する（APIキー・環境変数不要）
- 位置情報の設定は任意とする（未設定でも投稿可能）
- 地図クリックでピンを設置し、再クリックで上書き移動、✕ ボタンでリセットできる
- 住所はクライアント側の表示のみ。DB には lat/lng のみ保存する
- 地図表示はクライアントサイドコンポーネントとして実装する（`'use client'` が必要）

---

## 7. 対象外仕様

| 機能 | 理由 |
|------|------|
| ルート表示 | スコープ外 |
| GPS による現在地取得 | スコープ外 |
| オフライン地図 | スコープ外 |
