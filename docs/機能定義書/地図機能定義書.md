# 地図機能定義書

**バージョン:** 1.0
**作成日:** 2026-06-27
**更新日:** 2026-06-28

---

## 1. 機能概要

投稿した旅行スポットの位置情報を地図上に表示する機能。
Mapbox GL JS を使用し、投稿詳細画面にスポットの位置をピンで表示する。
地図ライブラリは Mapbox をデフォルトとするが、実装時に Leaflet + OpenStreetMap への変更も可能な構成にする。

---

## 2. ユーザーストーリー

| ID | ストーリー |
|----|-----------|
| US-01 | ユーザーとして、投稿詳細画面でスポットの場所を地図で確認したい |
| US-02 | ユーザーとして、投稿作成時にスポットの位置（緯度/経度）を設定したい |

---

## 3. 機能詳細

### 3.1 地図表示（投稿詳細）

- 投稿の `latitude` / `longitude` が設定されている場合のみ地図を表示する
- 未設定の場合は地図エリア自体を非表示にする
- Mapbox GL JS を使用してインタラクティブな地図を表示する
- スポットの位置にピンマーカーを表示する
- 初期ズームレベル：未定（実装時に決定）

### 3.2 位置情報の設定（投稿作成・編集）

- 入力方式：未定。以下の候補を実装時に選択する
  - **案 A**：緯度/経度の直接入力欄を設ける
  - **案 B**：地図上でクリックしてピンを立てる
  - **案 C**：場所名から Mapbox Geocoding API で自動取得する
- 位置情報の設定は任意とする（未設定でも投稿可能）

### 3.3 ライブラリ構成

| 項目 | 内容 |
|------|------|
| デフォルト | Mapbox GL JS |
| 代替候補 | Leaflet + OpenStreetMap（実装時に変更可能） |
| アクセストークン管理 | `NEXT_PUBLIC_MAPBOX_TOKEN` として環境変数で管理 |
| 無料枠 | 月 50,000 マップロードまで無料 |

---

## 4. 使用 API

| Method | エンドポイント | 説明 | 認証 |
|--------|--------------|------|------|
| （外部）| Mapbox GL JS | 地図タイル表示 | Mapbox トークン |
| （外部）| Mapbox Geocoding API | 場所名 → 緯度/経度変換（未定） | Mapbox トークン |

※ サーバーサイドの Route Handler は地図機能専用には設けない。座標データは投稿 API（`POST /api/posts`）で一緒に送受信する。

---

## 5. バリデーション仕様

| 項目 | 仕様 |
|------|------|
| 緯度 | 任意・-90 〜 90 の範囲（未定：実装時に検討） |
| 経度 | 任意・-180 〜 180 の範囲（未定：実装時に検討） |

---

## 6. 制約・前提条件

- 地図ライブラリの選定は Mapbox をデフォルトとするが、実装時に変更できる設計にする
- Mapbox アクセストークンは環境変数で管理し、ハードコードしない
- 位置情報の設定は任意とする
- 地図表示はクライアントサイドコンポーネントとして実装する（`'use client'` が必要）

---

## 7. 対象外仕様

| 機能 | 理由 |
|------|------|
| 複数スポットの地図一覧表示 | 未定（実装時に検討） |
| ルート表示 | スコープ外 |
| GPS による現在地取得 | スコープ外 |
| オフライン地図 | スコープ外 |
