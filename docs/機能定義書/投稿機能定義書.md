# 投稿機能定義書

**バージョン:** 1.1
**作成日:** 2026-06-27
**更新日:** 2026-06-28

---

## 1. 機能概要

旅行スポットの記録を投稿する機能。スポット名・感想テキスト・写真（複数枚）・エリアタグ・訪問日・カテゴリ・評価を 1 件の投稿として管理する。
写真は AWS S3 にアップロードし、URL を DB（PostImage テーブル）に保存する。
投稿は本人のみ編集・削除できる。

---

## 2. ユーザーストーリー

| ID | ストーリー |
|----|-----------|
| US-01 | ユーザーとして、訪れた旅行スポットを写真・感想・エリアタグ・訪問日・カテゴリ・評価とともに投稿したい |
| US-02 | ユーザーとして、投稿に複数枚の写真を添付したい |
| US-03 | ユーザーとして、自分の投稿を後から編集したい |
| US-04 | ユーザーとして、自分の投稿を削除したい |
| US-05 | ユーザーとして、投稿の詳細（写真・感想・コメント）を確認したい |
| US-06 | ユーザーとして、自分の投稿をどの画面からでも編集・削除したい |
| US-07 | ユーザーとして、旅行プランと投稿をリンクさせて費用を記録したい |

---

## 3. 機能詳細

### 3.1 投稿作成

- 入力項目：
  - スポット名（title）・感想（body）・エリアタグ（areaTag）
  - カテゴリ（category）：観光スポット / グルメ / 宿・ホテル / 自然 / アクティビティ / その他
  - 評価（rating）：星1〜5
  - 訪問日（visitedAt）：DATE 型。**必須入力**
  - 費用内訳（costBreakdown）：`[{ label: string, amount: number }]` の配列。自分のみ表示。合計を `cost` フィールドに保存
  - 写真（複数枚）
  - 緯度/経度（未定：地図機能と連携）
- 処理フロー：
  1. フロントエンドで写真を選択（複数枚）
  2. `POST /api/upload/post` で各写真を S3 にアップロード → URL を取得
  3. `POST /api/posts` で投稿データ + 画像URL配列を送信
  4. PostImage テーブルに displayOrder 順で保存
- 写真の制約：
  - 対応フォーマット：JPEG / PNG / WebP / GIF
  - 1枚あたり最大サイズ：未定（実装時に決定）
  - 最大枚数：未定（実装時に決定）

### 3.2 投稿一覧表示（フィード）

- フォロー中ユーザーの投稿を新着順に表示する
- 各投稿カードに表示する情報：
  - 投稿者のアバター・名前
  - スポット名（title）
  - 先頭の写真（PostImage の displayOrder=0）
  - エリアタグ・カテゴリ・評価（星）・訪問日
  - いいね数・コメント数
  - 投稿日時
- N+1 対策：`findMany` 時に `author`・`images`・`_count（likes/comments）` を `include` で一括取得する

### 3.3 投稿詳細表示

- 全写真をスライダー形式で表示する（order 順）
- 感想テキスト全文を表示する
- エリアタグ・カテゴリ・評価・訪問日・投稿日時を表示する
- いいね・コメント・行きたい・訪問済みの各ボタンを表示する
- 地図：緯度/経度が設定されている場合のみ Leaflet.js で表示
- 関連スポット：同エリアタグの他投稿をいいね数降順で最大3件表示（地図とコメントの間）

### 3.4 投稿編集

- 本人のみアクセス可能（他ユーザーが `/posts/[id]/edit` にアクセスした場合は 403）
- 既存の値をフォームにプリセットした状態で表示する
- 写真の追加・削除・並び替えが可能（未定：並び替えは実装時に検討）

### 3.5 投稿削除

- 本人のみ削除可能
- 削除時に S3 上の画像も削除する（未定：実装時に検討）
- 削除確認ダイアログを表示してから実行する

### 3.6 投稿の編集・削除ボタンの表示

- 自分の投稿カードには、どの一覧画面（ホーム・行きたい・フォロー中の投稿等）でもホバー時に ✏️（編集）・🗑️（削除）ボタンを表示する

## 4. 使用 API

| Method | エンドポイント | 説明 | 認証 |
|--------|--------------|------|------|
| GET | `/api/posts` | 投稿一覧取得（フィード） | 不要 |
| POST | `/api/posts` | 投稿作成 | 必要 |
| GET | `/api/posts/[id]` | 投稿詳細取得 | 不要 |
| PUT | `/api/posts/[id]` | 投稿編集 | 必要（本人のみ） |
| DELETE | `/api/posts/[id]` | 投稿削除 | 必要（本人のみ） |
| POST | `/api/upload/post` | 写真を S3 にアップロード | 必要 |

## 5. バリデーション仕様

| 項目 | フロントエンド | バックエンド |
|------|-------------|------------|
| スポット名（title） | 必須・1〜40文字・文字数カウンター表示 | 必須・1〜40文字 |
| 感想（body） | 必須・1文字以上 | 必須・1文字以上 |
| エリアタグ（areaTag） | 必須 | 必須・1〜50文字 |
| カテゴリ（category） | 必須・選択肢から選ぶ | 必須・許可値のみ受け付ける |
| 評価（rating） | 必須・1〜5 の整数 | 必須・1〜5 の整数のみ受け付ける |
| 訪問日（visitedAt） | **必須**・日付形式 | 必須・ISO 日付文字列 |
| 費用内訳（costBreakdown） | 任意・金額 > 0 またはラベルが入力済みの項目のみ保存 | 任意・配列形式 |
| 写真 | 任意・フォーマットチェック | 任意・MIME タイプ・サイズ検証 |

---

## 6. 制約・前提条件

- 写真の最大枚数・最大サイズは未定（実装時に決定）
- 地図機能（緯度/経度）との連携は地図機能定義書を参照
- 投稿はログインユーザーのみ作成・編集・削除できる

---

## 7. 対象外仕様

| 機能 | 理由 |
|------|------|
| 動画アップロード | スコープ外 |
| 写真の並び替え（ドラッグ&ドロップ） | 未定（実装時に検討） |
| 下書き保存 | スコープ外 |
