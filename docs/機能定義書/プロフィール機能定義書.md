**バージョン:** 1.1
**作成日:** 2026-06-27
**更新日:** 2026-06-28

## 1. 機能概要

ユーザーのプロフィール情報を表示・編集する機能。
プロフィール画面（公開）では投稿一覧・フォロワー/フォロー中数を確認できる。
訪問済み・行きたいリストはマイページ（非公開）で管理する。
プロフィール画像は当面ローカル保存し、AWS S3への移行は実装計画書 Phase 6 で行う。

---

## 2. ユーザーストーリー

| ID | ストーリー |
|----|-----------|
| US-01 | ユーザーとして、他ユーザーのプロフィール（ニックネーム・アイコン・自己紹介・投稿一覧）を確認したい |
| US-02 | ユーザーとして、自分のニックネーム・自己紹介を編集したい |
| US-03 | ユーザーとして、プロフィール画像をアップロード・変更したい |
| US-04 | ユーザーとして、マイページで自分の訪問済み / 行きたいリストを確認したい |
| US-05 | ユーザーとして、プロフィール画面でフォロワー数・フォロー中数を確認したい |
| US-06 | ユーザーとして、自分が投稿したコメントを一覧で確認したい |
| US-07 | ユーザーとして、自分の投稿に届いたコメントを一覧で確認したい（自分のみ閲覧可） |

## 3. 機能詳細

### 3.1 プロフィール表示

- URL：`/users/[id]`
- 表示情報：
  - プロフィール画像（未設定時はデフォルトアイコン）
  - ニックネーム（nickname）
  - 自己紹介（bio）
  - TabiScore（旅スコア・ランク）
  - 投稿数 / フォロワー数 / フォロー中数
  - フォローボタン（自分以外のユーザーに表示）
  - 投稿一覧タブ
  - 投稿したコメントタブ（対象ユーザーが書いたコメントの一覧）
  - 自分へのコメントタブ（**本人閲覧時のみ表示**：自分の投稿に届いたコメント一覧）
  - フォロワー一覧タブ
  - フォロー中一覧タブ
- 行きたい・訪問済みのカウントはプロフィール画面に表示しない（マイページで管理）

### 3.2 TabiScore（旅スコア）

ユーザーの旅行活動量を数値化したスコア。プロフィール画面に表示する。

**計算式**

```
TabiScore = 投稿数 × 10 + 訪問済み数 × 5 + 受けたいいね数 × 3 + 受けたコメント数 × 2
```

**ランク判定**

| スコア | ランク |
|-------|-------|
| 100 以上 | プラチナトラベラー |
| 60〜99 | ゴールドトラベラー |
| 30〜59 | シルバートラベラー |
| 0〜29 | ブロンズトラベラー |

- スコアとランクはリアルタイム集計（保存しない）
- プロフィール画面のヘッダーにバッジ形式で表示する

---

### 3.3 プロフィール編集

- URL：`/settings`（タイトル: ✏️ プロフィール編集）
- 自分のプロフィールのみ編集可能
- 編集可能項目：ニックネーム（nickname）・自己紹介（bio）
- `PUT /api/users/[id]` で更新する
- 保存成功後：トーストメッセージを表示し、自分の公開プロフィール画面へ遷移する

### 3.4 プロフィール画像アップロード（未実装、Phase 3-B予定）

- 設定画面（`/settings`）から画像を選択してアップロードする
- `POST /api/upload/avatar` でローカル保存し URL を取得（既存の `upload/post` と同じ方式を想定。S3移行はPhase 6）
- `PUT /api/users/[id]` で `image` URL を更新する
- 対応フォーマット：JPEG / PNG / WebP
- 最大サイズ：未定（実装時に決定）

## 4. 使用 API

**⚠️ 以下は全て未実装（Phase 3-B/3-C予定）。**

| Method | エンドポイント | 説明 | 認証 |
|--------|--------------|------|------|
| GET | `/api/users/[id]` | ユーザー情報取得 | 不要 |
| PUT | `/api/users/[id]` | プロフィール編集 | 必要（本人のみ） |
| POST | `/api/upload/avatar` | プロフィール画像をアップロード（Phase 3-B予定、ローカル保存） | 必要 |

**レスポンス例（ユーザー情報）**

```json
{
  "id": "cm_xxx",
  "nickname": "田中花子",
  "image": "https://s3.ap-northeast-1.amazonaws.com/tripdiary/...",
  "bio": "旅が好きです",
  "followerCount": 12,
  "followingCount": 8,
  "followedByCurrentUser": true,
  "postCount": 25
}
```

## 5. バリデーション仕様

| 項目 | フロントエンド | バックエンド |
|------|-------------|------------|
| ニックネーム（nickname） | 必須・1〜20文字・文字数カウンター表示 | 必須・1〜20文字 |
| 自己紹介（bio） | 任意・最大200文字・文字数カウンター表示 | 任意・最大200文字 |
| プロフィール画像 | 任意・フォーマットチェック | 任意・MIME タイプ・サイズ検証 |
| 他人のプロフィール編集 | - | 403 を返す |

---

## 6. 制約・前提条件

- プロフィール編集・画像アップロードはログインユーザーのみ、かつ本人のみ可能
- プロフィール表示は未認証ユーザーも閲覧可能（未定：実装時に確認）
- プロフィール画像の最大サイズは未定

---

## 7. 対象外仕様

| 機能 | 理由 |
|------|------|
| ユーザー名（@nickname）の設定 | スコープ外（name のみ管理） |
| アカウント削除 | スコープ外 |
| メールアドレス変更 | スコープ外 |
| パスワード変更 | スコープ外 |
