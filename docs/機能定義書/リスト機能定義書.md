## 1. 機能概要

投稿を「訪問済み」または「行きたい」としてリストに登録する機能。
いいねとは独立した 2 種類のリストを管理する。
登録・解除は各 API を呼び出すことでトグルする。

---

## 2. ユーザーストーリー

| ID | ストーリー |
|----|-----------|
| US-01 | ユーザーとして、すでに訪れたスポットを「訪問済み」としてマークしたい |
| US-02 | ユーザーとして、いつか行きたいスポットを「行きたい」リストに追加したい |
| US-03 | ユーザーとして、自分の訪問済み / 行きたいリストをマイページで確認したい |
| US-04 | ユーザーとして、リストへの登録を解除したい |

---

## 3. 機能詳細

### 3.1 訪問済みリスト

- `POST /api/posts/[id]/visited` を呼び出すことでトグル動作する
  - 未登録 → 訪問済みとして登録
  - 登録済み → 解除

### 3.2 行きたいリスト

- `POST /api/posts/[id]/wishlist` を呼び出すことでトグル動作する
  - 未登録 → 行きたいとして登録
  - 登録済み → 解除
- いいね機能とは別に管理する（いいね = 「行ってみたい」評価、行きたいリスト = 自分用のブックマーク）

### 3.3 リスト一覧表示（2026-07-20刷新）

- マイページ（`/mypage`）に「行きたい」「訪問済み」のタブで表示する（自分のみ閲覧可）。行きたい・訪問済みは同一コンポーネント（`SavedMapSection.tsx`）を`kind`プロパティで切り替えて共用する
- 上部に都道府県マップを表示し、登録件数が多い県ほど濃い色で塗り分ける（`intensityByLocation`）。マップの県をクリックすると該当県のグループ見出しへスムーズスクロールし、1.6秒間背景色でハイライトする
- マップの下に、`LOCATIONS`配列の並び順（北海道→沖縄県の地理順）で県ごとにグループ化したカード一覧を表示する。各グループ見出しに件数（「（{件数}件）」）を表示
- 各グループ内の投稿カードの並び順：登録日時の新着順（createdAt DESC）
- 地図が画面外にスクロールした場合、右下に「🗺️ 地図に戻る」の浮動ボタンを表示する（`IntersectionObserver`で地図の可視状態を判定）
- 公開プロフィール（`/users/[id]`）には表示しない

### 3.4 リストからの解除（一覧からの直接操作、2026-07-20追加）

- 各カードにホバー（PCの場合）またはタップで表示される「外す」アイコンボタンから、投稿を開かずにリスト解除できる（`RemovableSavedCard.tsx`）
- ボタン押下時は確認ダイアログ（「いつでもリストに戻せます。」）を表示し、確認後にトグルAPI（3.1/3.2節）を呼び出す
- 楽観的UI：確認と同時にカードを一覧から消し、あわせて所属する県グループの件数見出し・地図の濃淡もその場で再計算して反映する。API呼び出しが失敗した場合はカード・件数・地図濃淡のすべてを元の状態にロールバックし、エラートーストを表示する
- 成功時は`router.refresh()`でマイページの各タブのカウントバッジ（Server Component側の集計）を再取得する

### 3.4 ボタン表示仕様

| 種別 | 未登録 | 登録済み |
|------|--------|---------|
| 訪問済み | 「訪問済みに追加」ボタン（線のみ） | 「訪問済み」ボタン（塗りつぶし） |
| 行きたい | 「行きたい」ボタン（線のみ） | 「行きたい」ボタン（塗りつぶし） |

## 4. 使用 API

| Method | エンドポイント | 説明 | 認証 |
|--------|--------------|------|------|
| POST | `/api/posts/[id]/visited` | 訪問済み toggle（登録 / 解除） | 必要 |
| POST | `/api/posts/[id]/wishlist` | 行きたい toggle（登録 / 解除） | 必要 |
| GET | `/api/users/[id]` | 訪問済み / 行きたいリストを含むユーザー情報取得 | 不要 |

**レスポンス例（visited toggle）**

```json
{
  "visited": true
}
```

---

## 5. バリデーション仕様

| 項目 | 仕様 |
|------|------|
| 未認証ユーザーの操作 | 401 を返す |
| 重複登録 | toggle 処理のため重複は発生しない |
| 訪問済みと行きたいの両方への同時登録 | 制限なし（両方への同時登録を許容する） |

---

## 6. 制約・前提条件

- 訪問済みリストと行きたいリストは独立して管理する
- 両方のリストに同一投稿を登録することは許容する
- 登録はログインユーザーのみ操作可能

---

## 7. 対象外仕様

| 機能 | 理由 |
|------|------|
| リストの公開 / 非公開設定 | マイページ限定（自分のみ閲覧）に統一 |
| リストへのメモ追加 | スコープ外 |
| リストの共有 | スコープ外 |
