# 通知機能定義書

**バージョン:** 1.0
**作成日:** 2026-06-28
**更新日:** 2026-06-28
**作成者:** Nakata Saki

> ⚠️ **2026-07-08 更新：** いいね通知のアイコンを他画面のいいねボタン（❤️）と統一するため 👍 → ❤️ に変更（`NotificationList.tsx`）。

---

## 1. 機能概要

フォロー中のユーザーから受けたアクション（いいね・コメント・フォロー）を通知として一覧表示する機能。
未読件数をサイドバーのアイコンにバッジで表示し、通知アイテムがスクロールで画面内に入った時点で個別に既読になる（`IntersectionObserver` 方式）。

---

## 2. ユーザーストーリー

| ID | ストーリー |
|----|-----------|
| US-01 | ユーザーとして、自分の投稿にいいねされたことを通知で知りたい |
| US-02 | ユーザーとして、自分の投稿にコメントされたときにコメント内容を通知で確認したい |
| US-03 | ユーザーとして、誰かにフォローされたことを通知で知りたい |
| US-04 | ユーザーとして、未読通知がある場合にアイコンのバッジで気づきたい |

---

## 3. 機能詳細

### 3.1 通知の種別

| 種別 | アイコン | 生成タイミング | リンク先 |
|------|---------|-------------|---------|
| いいね（like） | ❤️ | 自分の投稿にいいねされたとき | 投稿詳細（S-04） |
| コメント（comment） | 💬 | 自分の投稿にコメントされたとき | 投稿詳細（S-04） |
| フォロー（follow） | 👤 | 自分がフォローされたとき | フォローしたユーザーのプロフィール（S-07） |

### 3.2 通知の表示内容

| 種別 | 表示テキスト |
|------|-----------|
| いいね | `{ユーザー名} が「{投稿タイトル}」にいいねしました` |
| コメント | `{ユーザー名} が「{投稿タイトル}」にコメントしました「{コメント本文冒頭}」` |
| フォロー | `{ユーザー名} があなたをフォローしました` |

- 各通知にアバター・相対日時を表示する
- 通知アイテムをクリックするとリンク先へ遷移する

### 3.3 未読バッジ・既読方式

- サイドバーの 🔔 通知アイコンに未読件数を赤バッジで表示する（TanStack Query の `refetchInterval: 60000` で60秒ごとにポーリング）
- 通知画面（`/notification`）で、各通知アイテムが `IntersectionObserver` によって画面内に入ったことを検知すると、その通知のみ個別に既読化する（`PATCH /api/notifications/[id]/read`）
- 画面を開いた瞬間に全件既読になるわけではなく、スクロールして表示された分だけ既読になる
- 既読処理は楽観的UI更新（即座にバッジを減算）＋失敗時のロールバックに対応

### 3.4 通知一覧

- 表示順：新しい通知が上（date DESC）
- 通知がない場合：「まだ通知はありません」メッセージを表示
- ページネーションは現時点では不要（全件表示）

## 4. データ構造（プロトタイプ）

通知はサーバーから取得するのではなく、`buildNotifications(userId)` 関数がローカルデータから動的に生成する。

```javascript
// 通知オブジェクトの構造
{
  type: 'like' | 'comment' | 'follow',
  fromUserId: string,   // 通知を発生させたユーザーのID
  post: Post | null,    // いいね・コメント時の対象投稿
  body: string | null,  // コメント時のコメント本文
  targetId: string | null, // フォロー時のフォロー対象ユーザーID
  date: string,         // 日時（ISO文字列）
  read: boolean,        // 既読フラグ
}
```

### 本実装時のデータモデル

| カラム | 型 | 説明 |
|-------|-----|------|
| id | VARCHAR(30) | 通知ID |
| userId | VARCHAR(30) | 通知を受け取るユーザーID（自分） |
| fromUserId | VARCHAR(30) | 通知を発生させたユーザーID |
| type | ENUM | `like` / `comment` / `follow` |
| postId | VARCHAR(30) | 対象投稿ID（いいね・コメント時） |
| commentBody | TEXT | コメント本文冒頭（コメント通知時） |
| read | BOOLEAN | 既読フラグ（デフォルト false） |
| createdAt | DATETIME | 通知生成日時 |

---

## 5. 使用 API

| Method | エンドポイント | 説明 | 認証 |
|--------|--------------|------|------|
| GET | `/api/notifications` | 通知一覧取得（新着順、最大50件） | 必要 |
| PATCH | `/api/notifications/[id]/read` | 通知を個別に既読化 | 必要（本人のみ） |
| GET | `/api/notifications/unread-count` | 未読件数取得（バッジ用） | 必要 |

---

## 6. バリデーション仕様

| 項目 | 仕様 |
|------|------|
| 自己通知 | 自分が自分の投稿にいいね・コメントしても通知は生成しない |
| 重複通知 | 同一ユーザーが同一投稿に複数回いいねしても通知は1件のみ生成する |

---

## 7. 制約・前提条件

- 通知は認証済みユーザーのみ閲覧可能
- 通知の保持期間：未定（実装時に決定）
- プッシュ通知（ブラウザ通知）は対象外

---

## 8. 対象外仕様

| 機能 | 理由 |
|------|------|
| プッシュ通知 | スコープ外 |
| メール通知 | スコープ外 |
| 通知の個別削除 | スコープ外 |
| 通知種別のフィルタリング | スコープ外 |
