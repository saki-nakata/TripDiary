## 1. 機能概要

他ユーザーをフォローする機能。
フォロー中のユーザーの投稿がフィードに表示される。
フォロー/アンフォローは同じ API を呼び出すことでトグルする。

---

## 2. ユーザーストーリー

| ID | ストーリー |
|----|-----------|
| US-01 | ユーザーとして、好きな旅行者をフォローしてフィードに投稿を表示したい |
| US-02 | ユーザーとして、フォローを解除したい |
| US-03 | ユーザーとして、他ユーザーのプロフィールでフォロワー数・フォロー中数を確認したい |
| US-04 | ユーザーとして、自分がフォローしているかどうかをプロフィール画面で確認したい |

---

## 3. 機能詳細

### 3.1 フォロー操作

- `POST /api/users/[id]/follow` を呼び出すことでトグル動作する
  - フォローしていない状態 → フォロー登録
  - フォロー済みの状態 → アンフォロー
- 処理はサーバーサイドで Prisma の upsert / delete で行う

### 3.2 ボタン表示仕様

| 状態 | 表示 |
|------|------|
| 未フォロー | 「フォロー」ボタン |
| フォロー済み | 「フォロー中」ボタン（hover 時「アンフォロー」に変化：未定） |
| 自分自身 | ボタン非表示 |

### 3.3 フォロワー / フォロー中数表示

- ユーザープロフィール画面（`/users/[id]`）に表示する
- フォロワー数・フォロー中数は `follows` テーブルから都度集計する（非正規化カウンタは未定）

### 3.4 フィードへの反映

- フィード（`/`）にはフォロー中ユーザーの投稿のみ表示する
- フォロー / アンフォロー後はフィードに即時反映する（未定：ページリロードで反映の可能性あり）

---

## 4. 使用 API

| Method | エンドポイント | 説明 | 認証 |
|--------|--------------|------|------|
| POST | `/api/users/[id]/follow` | フォロー toggle（登録 / 解除） | 必要 |
| GET | `/api/users/[id]` | フォロー状態・フォロワー/フォロー中数を含むユーザー情報取得 | 不要 |

**レスポンス例（POST /api/users/[id]/follow）**

```json
{
  "following": true
}
```
> フォロワー数・フォロー中数は含まない（件数を含めるのは `GET /api/users/[id]` のプロフィール取得側）。

---

## 5. バリデーション仕様

| 項目 | 仕様 |
|------|------|
| 自己フォロー | 400 を返す（DB のチェック制約でも保証） |
| 未認証ユーザーのフォロー | 401 を返す |
| 重複フォロー | toggle 処理のため重複は発生しない |

---

## 6. 制約・前提条件

- 自分自身はフォローできない
- フォローはログインユーザーのみ操作可能

---

## 7. 対象外仕様

| 機能 | 理由 |
|------|------|
| フォロー中ユーザー一覧ページ | Phase 3-C で実装済み（`/users/[id]` の「フォロワー」「フォロー中」タブ）。専用の独立ページとしては作らずプロフィール画面内のタブに統合 |
| フォロー申請（プライベートアカウント） | スコープ外 |

> ⚠️ **2026-07-05 更新：** フォロー通知は `notification.service.ts` に `type: "follow"` として実装済み（Phase 2-E）。以前は「スコープ外」としていたが誤りだったため削除。
