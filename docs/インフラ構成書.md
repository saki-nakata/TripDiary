## 1. インフラ概要

### 設計方針

- サーバー管理不要の PaaS（Vercel・Railway）を採用し、インフラ運用コストを最小化する
- 無料枠の範囲内で運用する
- インフラのコード管理（Terraform 等）は未定

---

## 2. アーキテクチャ図

```
[ユーザー（ブラウザ）]
        ↕ HTTPS
[Vercel（Next.js アプリ）]
  ├── フロントエンド（App Router / RSC）
  └── バックエンド（Route Handlers）
        ↕ Prisma（TCP / SSL）
[Railway（MySQL）]
        ↕ Cloudinary API（HTTPS）
[Cloudinary（画像ストレージ）]
        ↕ Mapbox API（HTTPS）
[Mapbox（地図サービス）]
```

### 3.2 Railway（MySQL）

| 項目 | 内容 |
|------|------|
| 用途 | MySQL データベースのホスティング |

| 項目 | 内容 |
|------|------|

| 画像変換 | 未定（サムネイル生成等は実装時に検討） |
| 自動削除 | 未定（投稿削除時の Cloudinary 画像削除は実装時に検討） |

### 3.4 Mapbox（地図サービス）

| 項目 | 内容 |
|------|------|
| 用途 | 旅行スポットの位置を地図で表示 |
| プラン | Free（月 50,000 マップロードまで） |
| ライブラリ | Mapbox GL JS（代替候補：Leaflet + OpenStreetMap） |
| アクセストークン管理 | `NEXT_PUBLIC_MAPBOX_TOKEN` 環境変数で管理 |

## 4. 環境変数

| 変数名 | 設定先 | 説明 |
|--------|--------|------|
| `DATABASE_URL` | Vercel 環境変数 / `.env.local` | MySQL 接続 URL（Railway） |
| `NEXTAUTH_SECRET` | Vercel 環境変数 / `.env.local` | Auth.js のシークレットキー |
| `NEXTAUTH_URL` | Vercel 環境変数 / `.env.local` | アプリの URL |
| `CLOUDINARY_CLOUD_NAME` | Vercel 環境変数 / `.env.local` | Cloudinary クラウド名 |
| `CLOUDINARY_API_KEY` | Vercel 環境変数 / `.env.local` | Cloudinary API キー |
| `CLOUDINARY_API_SECRET` | Vercel 環境変数 / `.env.local` | Cloudinary API シークレット |
| `NEXT_PUBLIC_MAPBOX_TOKEN` | Vercel 環境変数 / `.env.local` | Mapbox アクセストークン（クライアントサイド） |
