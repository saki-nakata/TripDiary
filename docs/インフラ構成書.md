## 1. インフラ概要

### 設計方針

- AWS を中心としたクラウド構成で統一する
- EC2 で Next.js アプリを稼働、RDS で MySQL を管理、S3 で画像を保管
- 無料枠（Free Tier）の範囲内での運用を目指す
- インフラのコード管理（Terraform 等）は未定

---

## 2. アーキテクチャ図

```
[ユーザー（ブラウザ）]
        ↕ HTTPS
[EC2（Next.js アプリ / Nginx + PM2）]
  ├── フロントエンド（App Router / RSC）
  └── バックエンド（Route Handlers）
        ↕ Prisma（TCP / SSL）
[RDS（MySQL）]
        ↕ AWS SDK（HTTPS）
[S3（画像ストレージ）]
        ↕ Mapbox API（HTTPS）
[Mapbox（地図サービス）]
```

| 変数名 | 設定先 | 説明 |
|--------|--------|------|

## 5. デプロイ手順

### 初回セットアップ（未定：詳細は実装時に確定）

```bash
# 1. AWS コンソールで EC2 インスタンスを作成（t2.micro / Amazon Linux 2023）

# 2. EC2 に Node.js・Nginx・PM2 をインストール
sudo dnf install -y nodejs nginx
pnpm add -g pm2

# 3. RDS で MySQL インスタンスを作成し DATABASE_URL を取得

# 4. S3 バケットを作成し IAM ユーザー（アクセスキー）を発行

# 5. Mapbox アカウントを作成しアクセストークンを取得

# 6. EC2 に .env.local を配置し環境変数を設定

# 7. DB マイグレーション（初回）
npx prisma migrate deploy

# 8. PM2 でアプリ起動
pm2 start pnpm --name "tripdiary" -- start
pm2 startup
pm2 save
```

### 通常デプロイ

```bash
# 未定（GitHub Actions + SSH でのデプロイ等を実装時に検討）
```

## 7. セキュリティ方針

| 項目 | 方針 |
|------|------|
| パスワード | bcrypt（ソルトラウンド：12）でハッシュ化 |
| セッション | Auth.js JWT Strategy（HttpOnly Cookie に JWT を保存、DB への書き込みなし） |
| 環境変数 | .env.local はリポジトリにコミットしない（.gitignore で除外） |
| AWS IAM | S3 アクセス用 IAM ユーザーは最小権限ポリシーを付与 |
| AWS IAM キー | `AWS_SECRET_ACCESS_KEY` はサーバーサイドのみで使用（クライアントに公開しない） |
| S3 バケット | パブリックアクセス設定・バケットポリシーは実装時に確定 |
| HTTPS | 未定（Let's Encrypt / ACM + ALB 等） |
| セキュリティグループ | EC2：80/443 を公開、3306（MySQL）は EC2 からのみ許可 |
| CORS | 未定（実装時に検討） |
| レート制限 | 未定（実装時に検討） |
