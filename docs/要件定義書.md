## 1. プロジェクト概要

### 1.1 プロジェクト名

TripDiary

### 1.2 目的

旅行スポットを記録・共有できる SNS 型旅行記録アプリを作成する。
複数ユーザーが利用し、投稿・コメント・いいね・フォローなどのソーシャル機能を通じてユーザー同士が旅行情報を共有できる Web アプリケーションを提供する。

### 1.3 背景

プログラミングスクール中級編の最終課題として開発する。
Next.js・Prisma・MySQL・AWS（EC2・RDS・S3）を組み合わせ、フロントエンドからインフラまで一連の Web アプリケーション開発の流れを体験することを目的とする。

### 1.4 対象ユーザー

- 旅行好きの個人ユーザー
- 旅行情報を記録・共有したいユーザー
- スマートフォン・PC の両方から利用するユーザー

## 2. システム概要

### 2.1 システム構成

```
[ユーザー（ブラウザ）]
        ↕ HTTPS
[EC2（Next.js アプリ / Nginx + PM2）]
  ├── フロントエンド（App Router / React Server Components）
  └── バックエンド（Route Handlers / API）
        ↕ Prisma（TCP / SSL）
[RDS（MySQL）]
        ↕ AWS SDK（HTTPS）
[S3（画像ストレージ）]
        ↕ Leaflet + OpenStreetMap
[OpenStreetMap（地図サービス）]
```

### 2.2 技術スタック

| 役割 | 技術・バージョン |
|------|----------------|
| フロントエンド | Next.js 16.2.9 + TypeScript |
| スタイリング | Tailwind CSS 4.3.1 |
| バックエンド | Next.js Route Handlers |
| ORM | Prisma 7.8.0 |
| 認証 | Auth.js（next-auth 5.0.0-beta.31） |
| データベース | MySQL（RDS） |
| 画像ストレージ | AWS S3 |
| 地図 | Leaflet + OpenStreetMap |
| ホスティング | AWS EC2（アプリ）+ AWS RDS（MySQL） |

※ インフラの詳細は [インフラ構成書.md](インフラ構成書.md) を参照。

## 4. 非機能要件

| 分類 | 要件 |
|------|------|
| レスポンシブ対応 | スマートフォン・タブレット・PC のいずれでも正常動作すること |
| パフォーマンス | フィード初期表示は 3 秒以内 |
| セキュリティ | パスワードは bcrypt でハッシュ化して保存する |
| セキュリティ | 認証が必要な API はセッション検証を行う |
| セキュリティ | アップロード画像の MIME タイプ・サイズを API 側で検証する |
| セキュリティ | Prisma のパラメータバインディングにより SQL インジェクションを防止する |
| 可用性 | AWS（EC2・RDS・S3）の Free Tier の範囲内で動作すること |

## 5. 制約・前提条件

| 分類 | 内容 |
|------|------|
| インフラ | AWS（EC2・RDS・S3）の Free Tier の範囲内で構築する（Free Tier 終了後は費用が発生する） |
| 地図 | Leaflet + OpenStreetMap を使用する（無料・APIキー不要） |
| 認証 | Auth.js v5（beta）を使用するため、破壊的変更のリスクがある |
| HTTPS | 未定（Let's Encrypt / ACM + ALB 等で対応予定） |

## 7. 用語定義

| 用語 | 説明 |
|------|------|
| スポット | ユーザーが記録する旅行先の場所。1 件の投稿が 1 スポットに相当する |
| 投稿 | スポット名・感想テキスト・写真（複数枚）・エリアタグで構成される 1 件のデータ |
| 探索 | 全ユーザーの投稿を未認証でも閲覧できる公開フィード画面（`/`）。新着順または人気順で表示 |
| フォロー中の投稿 | ログインユーザーがフォロー中のユーザーの投稿を新着順に一覧表示するタブ（マイページ内） |
| 旅行プラン | 旅行の計画を管理するデータ。タイトル・日程・スポット・予算内訳・メモで構成される（作成者のみ閲覧） |
| TabiScore | 投稿数・訪問済み数・いいね数・コメント数から算出するユーザーの旅行活動スコア |
| 費用内訳 | 投稿に記録する旅の費用（項目名・金額の配列）。自分のみ表示される |
| 予算内訳 | 旅行プランに記録する予算（項目名・金額の配列） |
| カテゴリ | スポットの種別。観光 / グルメ / 宿・ホテル / 自然 / アクティビティ / 歴史・文化 / その他 |
| 評価 | 投稿者によるスポットの満足度（星1〜5） |
| 訪問日 | スポットを訪れた日付（visitedAt）。投稿日とは別に記録する |
| いいね | ユーザーが投稿に対して行う「行ってみたい」評価アクション。1 ユーザー 1 投稿につき 1 回のみ |
| コメント | 投稿に対して他のユーザーが追加するテキストの返信 |
| フォロー | 特定のユーザーの投稿をフィードに表示するためのアクション |
| フォロワー | 自分のことをフォローしているユーザー |
| フォロー中 | 自分がフォローしているユーザー |
| エリアタグ | 投稿に付与する地域・エリアを示す文字列（例：北海道、京都、沖縄） |
| 訪問済みリスト | ユーザーが「訪問した」としてマークした投稿の一覧 |
| 行きたいリスト | ユーザーが「行きたい」としてマークした投稿の一覧（いいねとは別の機能） |
| Route Handlers | Next.js App Router におけるサーバーサイド API の実装方式 |
| Auth.js | next-auth v5。Next.js に特化した認証ライブラリ（セッションベース） |
